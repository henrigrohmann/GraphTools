# 結合テスト チェックリスト

フロント・バック統合テストの確認事項を記載します。  
以下の項目を順に確認してください。

## セットアップ確認

- [ ] VS Code: **Terminal** → **Run Task** で **"Start FastAPI (uvicorn)"** を実行
  - ログに `INFO:     Uvicorn running on http://0.0.0.0:8005` が表示される
  - ターミナルタブに背景があり、プロセスが running 状態

- [ ] **"Start Frontend Static Server"** を実行
  - ログに `Serving HTTP on ...` が表示される

- [ ] ブラウザで `http://localhost:8002` を開く
  - タイトル: "GraphTool"
  - 上部: ボタン群 + "API Base" 表示
  - 中央: 散布図エリア（最初は空または初期状態）
  - 右: タブ（詳細 / 階層ビュー）
  - 下: 緑文字のログパネル + 現在地表示

## 機能テスト

### T1: ヘルスチェック

**操作**: [ヘルスチェック] ボタンをクリック

**確認項目**:
- [ ] ログパネルに時刻付きメッセージが表示される
- [ ] "HEALTH OK ..." または "HEALTH NG ..." が表示
- [ ] NG の場合、詳細パネルにエラー内容が表示される

**パス条件**: "HEALTH OK" が表示される

---

### T2-1: 散布図表示（生データ）

**操作**: [生データ] ボタンをクリック

**予想ログ**:
```
[HH:MM:SS] FETCH http://...api/raw
[HH:MM:SS] STATUS 200 /raw
[HH:MM:SS] FETCH http://...api/scatter?mode=raw
[HH:MM:SS] STATUS 200 /scatter?mode=raw
[HH:MM:SS] SCATTER READY count=XXX
```

**UI確認**:
- [ ] 散布図エリアに青い点群が表示される
- [ ] 点の数が "SCATTER READY count=N" と一致

**パス条件**: 点が 1 個以上表示される

---

### T2-2: 散布図表示（ランダム）

**操作**: [ランダム] ボタンをクリック

**確認項目**:
- [ ] 散布図が再描画される
- [ ] 点の配置が「生データ」と異なる（ランダムなはず）
- [ ] ログに "SCATTER READY" が出現

**パス条件**: ランダムな点群が表示される

---

### T2-3: 散布図表示（クラスタリング）

**操作**: [クラスタリング] ボタンをクリック

**確認項目**:
- [ ] 散布図が再描画される
- [ ] 点が幾何学的に集約された配置に見える
- [ ] ログに "SCATTER READY" が出現

**パス条件**: クラスタ化された点群が表示される

---

### T2-4: 散布図表示（濃い意見）

**操作**: [濃い意見] ボタンをクリック

**確認項目**:
- [ ] 散布図が再描画される
- [ ] 全体的に左下など特定エリアに集中した配置が見える（密度フィルタ）
- [ ] ログに "SCATTER READY" が出現

**パス条件**: 密度フィルタ後の点群が表示される

---

### T3: 詳細情報表示

**操作**: T2 で表示した散布図の点を **1つクリック**

**確認項目**:
- [ ] 右パネル「詳細」タブがアクティブになる
- [ ] 以下が表示される:
  - `ID: <id_value>`
  - `内容: <text_value>`
- [ ] テキストは HTML エスケープされている（`<` が `&lt;` など）

**パス条件**: ID と内容が正しく表示される

---

### T4: 階層ビュー表示

**操作**: 
1. [クラスタリング] をクリック（散布図読み込み）
2. 右パネルの「階層ビュー」タブをクリック

**確認項目**:
- [ ] タブが切り替わる
- [ ] クラスタリスト（例: `cluster:A`, `cluster:B`, `cluster:C`）が表示
- [ ] 各クラスタの下に、意見が `• <opinion>` 形式で箇条書きされている
- [ ] ログに "HIERARCHY READY" が出現

**パス条件**: クラスタと意見の階層が表示される

---

### T5: API テスター（詳細確認用）

**操作**: 
1. ブラウザで `http://localhost:8002/api_tester.html` を開く
2. **[Run All]** をクリック

**確認項目**:
- [ ] 結果テーブルに 必須テスト X 個が表示される
- [ ] すべてのテストが **PASS** （緑色）

**パススコア**: すべてのテストが PASS

## NG 時のチェック

### 散布図が出ない場合

1. **ログパネルを確認**
   - HTTP エラー番号が出ているか
   - 404 → エンドポイント未実装
   - 500 → バックエンド処理エラー

2. **API Base が正しいか**
   - `http://127.0.0.1:8005` (ローカル)
   - `https://xxxxx-8005.app.github.dev` (Codespaces)

3. **パイプラインが実行されているか**
   - [ヘルスチェック] → "jobs=N" の数が増えているか確認

### 詳細情報が表示されない場合

1. 点をクリックしたか（ダブルクリックではない）
2. 詳細タブが実際に切り替わったか

### 階層ビューが空の場合

1. クラスタリングパイプラインが実施されたか
2. ログで "HIERARCHY READY" が表示されているか

## 備考

- 各テストは**独立**しており、前のテスト結果に依存しません
- ファイルの大きさにより、処理時間は数秒～数十秒になる場合があります
- Codespaces ハング時は、該当タスクを停止して再度起動してください
